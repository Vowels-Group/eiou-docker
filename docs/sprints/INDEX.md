## How to Use This System

### Creating a New Sprint

1. **Copy the template**:
   ```bash
   cp docs/sprints/TEMPLATE.md docs/sprints/SPRINT-$(date +%Y%m%d)-description.md
   ```

2. **Fill in sprint metadata**:
   - Sprint ID, dates, branch name
   - Primary objectives and success criteria

3. **Update INDEX.md**:
   - Add new sprint to "Active Sprint" section
   - Move previous sprint to "Completed Sprints"

4. **Update README.md**:
   - Update "Recommended Next Steps" section
   - Update "Current Sprint" reference

### During the Sprint

- Update tasks as they're completed
- Document challenges and solutions
- Track metrics and performance
- Commit sprint doc updates regularly

### Completing a Sprint

1. Fill out retrospective section
2. Add final metrics and outcomes
3. Move sprint from "Active" to "Completed" in INDEX.md
4. Update sprint statistics
5. Create next sprint document
