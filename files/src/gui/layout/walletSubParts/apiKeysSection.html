About API Keys
API keys let external tools — scripts, accounting importers, monitoring dashboards — talk to this node over HTTPS without sharing your auth code. Each key carries a fixed subset of permissions (scopes) granted at creation time and cannot be upgraded afterwards — create a new key instead.

Label is for you — something that will help you recognise what the key is for in the list later. Presets fill the permission checkboxes with common bundles (Read-only, Full access); you can always tick or untick individual scopes afterwards.

Rate limit caps how many requests per minute the key is allowed; default 100, max 1000. Expires sets a hard cut-off after which the key is rejected — for scripts that should never live forever. Pick Never for keys you manage lifecycle of yourself.

The secret is shown once on the next screen and never again. Copy it somewhere safe before closing the modal.
