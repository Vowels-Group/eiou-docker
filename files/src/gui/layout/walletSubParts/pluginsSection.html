Plugins

Extend your node with optional add-ons
Plugins live under /etc/eiou/plugins/. They subscribe to internal events (sync, transactions, contacts) and add capabilities without modifying the core code.

Plugins are disabled by default. A new plugin sitting in the directory will not run until you enable it here and restart the node. This way a buggy or untrusted plugin cannot crash the node on its first boot — you opt in deliberately, with the chance to read the manifest first. Toggles persist immediately but take effect only after a restart, since event subscriptions are wired during startup.
Plugin changes are saved. Restart the node for them to take effect.
