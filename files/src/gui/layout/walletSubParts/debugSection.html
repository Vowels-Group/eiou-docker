About running processes
The EIOU log tags each entry with [PID:N]. Use this table to identify which long-running processor that PID belongs to.

Short-lived P2P worker children (forked per outgoing message) are intentionally not listed — they exit in seconds and almost never appear in the log lookup window.

Status meanings: Running = lockfile exists and the PID is alive; Stale lockfile = lockfile holds a PID that no longer exists (usually a crashed processor; the watchdog cleans these on the next restart); Stopped = no lockfile.
